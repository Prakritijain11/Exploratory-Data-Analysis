# Vendor Performance Analysis

An end-to-end data pipeline and exploratory analysis that turns raw retail
transaction data (purchases, sales, purchase prices, and vendor invoices) into
an analysis-ready **vendor–brand performance summary**, then mines it for
actionable insights on profitability, purchasing concentration, bulk-pricing,
and slow-moving inventory.

## Pipeline overview

```
data/*.csv ──▶ ingestion_db.py ──▶ inventory.db (raw tables)
                                        │
                                        ▼
                              get_vendor_summary.py
                                        │
                                        ▼
                        vendor_sales_summary  (cleaned + KPIs)
                                        │
                                        ▼
                    Vendor_Performance_Analysis.ipynb (insights)
```

## Repository contents

| File | What it does |
|------|--------------|
| `ingestion_db.py` | Reads every CSV in `data/` and loads each into `inventory.db` as its own table (via SQLAlchemy, `if_exists='replace'`). Logs progress and total runtime to `logs/ingestion.log`. |
| `get_vendor_summary.py` | Builds the `vendor_sales_summary` table: a single SQL query (CTEs for freight, purchases, and sales) produces one row per **vendor–brand**; the data is then cleaned and enriched with KPIs and written back to `inventory.db`. Logs to `logs/Get_vendor_summary.log`. |
| `Exploratory_Data_Analysis.ipynb` | The exploration that shaped the summary query — inspects the raw tables, prototypes the joins, cleans the data, engineers the metrics, and persists the summary table. |
| `Vendor_Performance_Analysis.ipynb` | The analysis on the summary table — distributions, correlations, brand/vendor rankings, Pareto concentration, bulk-purchase pricing, unsold-inventory capital, and a confidence-interval comparison of top vs. low vendors. |

## The `vendor_sales_summary` table

`get_vendor_summary.py` consolidates four raw tables into one row per
vendor–brand pairing:

- **Source join** — `PurchaseSummary` (purchases ⨝ purchase_prices on `Brand`,
  keeping only `PurchasePrice > 0`) is `LEFT JOIN`ed to `SalesSummary`
  (on vendor + brand) and to `FreightSummary` (on vendor).
- **Cleaning** (`clean_data`) — casts `Volume` to float, fills missing
  sales-side values with 0, and strips whitespace from `VendorName` /
  `Description`.
- **Engineered KPIs:**
  - `Gross_Profit` = `TotalSalesDollars − TotalPurchaseDollars`
  - `ProfitMargin` = `Gross_Profit / TotalSalesDollars × 100`
  - `StockTurnover` = `TotalSalesQuantity / TotalPurchaseQuantity`
  - `SalestoPurchaseRatio` = `TotalSalesDollars / TotalPurchaseDollars`

## Getting started

### Prerequisites

```bash
pip install pandas numpy matplotlib seaborn scipy sqlalchemy
```

### Run the pipeline

1. Place the source CSVs in a `data/` folder (e.g. `purchases.csv`,
   `purchase_prices.csv`, `sales.csv`, `vendor_invoice.csv`). The table name is
   taken from each file name.

2. Ingest the raw data into `inventory.db`:
   ```bash
   python ingestion_db.py
   ```

3. Build the cleaned vendor summary table:
   ```bash
   python get_vendor_summary.py
   ```

4. Open `Vendor_Performance_Analysis.ipynb` to explore the insights (the
   `logs/` folder is created automatically).

## Key analyses in the notebook

- **Promising brands** — low sales volume but high profit margin (candidates for
  promotion or a pricing review).
- **Vendor concentration** — a Pareto chart showing how much of total purchase
  spend the top 10 vendors account for (dependency risk).
- **Bulk purchasing** — whether larger order sizes lower the average unit price.
- **Slow-moving inventory** — vendors with `StockTurnover < 1` and the capital
  locked in unsold stock.
- **Top vs. low vendors** — 95% confidence intervals testing whether the two
  groups' profit margins differ meaningfully.
